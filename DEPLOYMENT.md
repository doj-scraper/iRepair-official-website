Build & deploy (quick)

1. Install deps: npm ci
2. Build: npm run build
3. Deploy: connect this repository to Vercel (recommended) or use any static hosting; set environment variables in the platform:
   - VITE_SUPABASE_URL
   - VITE_SUPABASE_PUBLISHABLE_KEY
   - DATABASE_URL (if migrations/Prisma are run)
   - Optional: NEON_API_KEY (for provisioning Neon DB)

Vercel quick steps:
- Create a GitHub repo for this project and push main branch.
- In Vercel, Import Project -> select GitHub repo -> set environment variables -> Deploy.
- Vercel will run: npm ci && npm run build by default.

CI & tests

- Unit tests: npm run test:unit (Vitest) — unit tests added under src/__tests__/
- E2E tests: tests/e2e/ (Playwright) — requires browsers: npx playwright install and a running dev server or preview URL.
- To run both locally: npm run test:unit && (start dev server) && npx playwright test

Playwright notes

Neon DB provisioning

- Provisioning a secondary Neon DB requires a Neon API token (NEON_API_KEY) with project creation permissions.
- Example curl to create a project (replace NEON_API_KEY and NEON_ORG_ID):
  curl -X POST "https://api.neon.tech/v1/projects" \
    -H "Authorization: Bearer $NEON_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{"name":"irepair-connect-secondary-2026-04-25","org_id":"'$NEON_ORG_ID'","region":"us-west-2"}'
